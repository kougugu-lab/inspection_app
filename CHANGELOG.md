# CHANGELOG

本アプリケーションの変更履歴を記録します。

## [2026-04-09] v3.3.2 - サイクル処理の堅牢化と操作性改善
### 修正
- **コミット番号更新ロジックの強化**: 複数トリガー構成やノイズによるシーケンスの不整合を防ぐため、物理的な最終トリガー検知による強制サイクル完了ロジックを導入。OK/NGに関わらず確実にカウントアップされるよう改善しました。
- **設定・テンキー画面の操作性改善 (Wayland対策)**: Raspberry Pi OS (Bookworm) のデフォルト環境において設定画面等がクリックに反応しない問題を解消するため、`focus_force` と遅延 `grab_set` によるフォーカス制御を最適化。
- **CSV出力形式の修正**: 判定条件や検出結果にカンマが含まれる場合にExcelで列がずれる問題を修正。標準の `csv` モジュールを使用し、適切にクォート処理を行うようにしました。
- **ブザー停止時の例外修正**: Lazy Load設定時にブザー停止ボタンを押すと `pygame.error: mixer not initialized` が発生する問題を修正。
- **ログ視認性の向上**: アプリログ出力時、内部ID（pat_1等）ではなく、ユーザーが設定した表示名を使用するように改善。

## [2026-04-08] v3.3.1 - プレビュー映像の最適化・起動時間短縮・不具合修正
### 修正
- **プレビュー映像のさらなる軽量化**: Raspberry Pi 5での描画カクつきを最小限にするため、描画が追いつかない場合に画像のリサイズ・色変換処理そのものをスキップする動的間引き処理を強化。CPU負荷を大幅に削減しました。
- **プレビュー映像の遅延解消**: OpenCVの仕様により生じていた映像の遅延・ラグを解消するため、内部バッファサイズを強制的に1 (`CAP_PROP_BUFFERSIZE = 1`) に設定。
- **インポートエラーの修正**: 内部で使用していた `Path` (pathlib) のインポート漏れ、および一部環境で発生していた `COLOR_HIGHLIGHT` の参照エラーを修正。

### 最適化
- **起動時間の短縮**: 起動時の待機時間を減らすため、以下の処理を改善しました。
  - **カメラの並列オープン**: 複数カメラの初期化を並列スレッドで同時実行し、起動待ち時間を短縮。
  - **pygame遅延初期化**: `pygame.mixer.init()` を起動時ではなく、音声を初めて再生する時点まで遅延。

## [2026-04-05] v3.3.0 - コードレビューに基づく安定性向上と設定画面のUI刷新
### 修正 (コードレビューによる改善)
- **容量監視・ディスク計算の非同期化**: ストレージ走査や使用量計算をバックグラウンドスレッドへ移行。画像が大量にある環境でもGUIがフリーズしないよう改善。
- **モデルロードの最適化**: クラス一覧取得時にロード済みのモデルを再利用（または必要時のみ再走査）するようにし、メモリ消費を抑制。
- **保存権限チェックの追加**: 設定保存時、出力先への書き込みテストを実施し、不備があれば警告。
- **コード・インポートの整理**: 重複コードの排除、インポートのモジュール上部集約、エラーハンドリングの適正化を全面的に実施。
- **マウスホイール挙動の改善**: ヘルプ画面等のスクロールイベントが他のウィンドウに干渉しないよう、フォーカス制御を厳密化。

### 変更
- **GPIOテスト画面の日本語化**: GPIOTestDialog の英語表記を日本語（トリガー等）に統一。
- **未使用定数の削除**: 不要となった `COLOR_HIGHLIGHT` を削除。

## [2026-03-29] v3.2.0 - 設定画面のハイライト機能刷新とプロジェクト構造の整理
### 修正
- **UI即時反映の修正**: 「トリガー別判定条件」で条件を追加・削除した際、画面に即座に反映されない不具合を修正。追加・削除後にパターン描画全体を再実行（`on_pat_sel`）することで、スクロール領域を含めた確実な描画更新を実現しました。
- **スコープ問題の根本修正**: Pythonのループ処理（カメラ、GPIO、パターン設定）において、`trace_add` や `lambda` がループの最後の変数値を参照してしまうスコープバグを修正。各行の生成処理を独立した関数に分離することで、データの独立性と保存の確実性を向上させました。
- **データ不整合の再発防止**: 追加・削除操作時に前の行のクラス名が空欄に戻ってしまうなどの不整合を完全に解消しました。

### 変更
- **設定画面のハイライト機能刷新**: 
  - 個別の入力項目がオレンジ色に変わる「変更点ハイライト」を廃止し、画面全体のノイズを低減しました。
  - 代わりに OCR_app 方式を採用し、**設定に変更があった場合のみ「保存して閉じる」ボタンが緑色（COLOR_OK）に変わり、テキストが「変更を適用して保存」に更新される**仕組みを導入。
  - テキスト入力、数値変更、チェックボックス、パターンの追加・削除のすべてがボタンの色変化に連動するように統一しました。
- **UI配色調整**: キャンセルボタンの色を `#546E7A` (Blue Grey) に変更し、保存ボタンとの視覚的な区別を明確にしました。

- **プロジェクト構造の整理**: 
  - プロジェクト構成を整理するため、`main.py` を除くすべてのソースコード (`app.py`, `dialogs.py`, `hardware.py`, `settings.py`, `widgets.py`, `constants.py`) を `modules/` フォルダへ移動しました。
  - インポートパスを `modules` パッケージ経由に統一し、モジュール管理を適正化しました。
- **設定ファイルの名称簡略化**:
  - 設定ファイルの名称を `inspection_settings_v3.2.json` から `inspection_settings.json` へ変更し、バージョン管理に依存しない固定名称としました。
  - `constants.py` 内の `SETTINGS_FILE` 指定も自動更新し、シームレスに移行しました。

## [2026-03-28] v3.1.0 - 判定ロジックの厳格化とUIの完全日本語化
### 修正
- **データ不整合の修正**: パターン設定で条件を追加・削除した際、入力中のクラス名が消失したり、タブを切り替えないと反映されない不全を修正。`.setdefault()` によるデータ参照の適正化により、操作が即座に反映されるようになりました。
- **IndexErrorの修正**: 判定条件を削除する際に稀に発生していた `pop index out of range` エラーを、リスト要素の直接削除方式に変更することで解消。
- **表記残りの修正**: カメラ設定カードのタイトル等に残っていた `Camera` `Index` 表記を、すべて日本語（カメラ、インデックス）に置換。
- **内部エラー修正**: 設定画面での `NameError` や特定条件下でのトリガー描画エラー、インデントミス等を全面的に修正。
- **GPIO表示復旧**: 編集ミスにより一時的に消えていた GPIO 入力状態を示す LED インジケータを再表示。

### 追加
- **ツールチップの拡充**: カメラ設定、GPIO設定、パターン設定の全主要項目に対して、操作ガイドとなるツールチップを配置。

### 変更
- **判定ロジックの厳格化**: 
  - 判定条件の **「基準個数」を一致判定（Exactly X）に変更**。基準個数より多い、または少ない場合は NG と判定されるようになりました。
  - 基準個数の空欄を禁止し、設定漏れがある場合は NG と判定するように変更。
- **完全日本語化 (カタカナ化)**: 
  - 設定画面、メイン画面、ログ出力の全項目において、技術用語を適切な日本語に置換。
  - `Index` → `インデックス`、`Camera` → `カメラ`、`Pin` → `ピン`、`Trigger` → `トリガー` 等。
- **UI改善**:
  - ツールチップの表示位置を「項目名（ラベル）」に統一（画素数タブの様式に準拠）。
  - ヘッダー部分などの技術的な補足表記を削除し、日本語のみのシンプルな表示にクリーンアップ。
- **パフォーマンス最適化**: Raspberry Pi 5 での動作軽量化のため、設定画面表示中はメインのカメラプレビュー処理を一時停止する仕組みを導入。
