# AI 自動検査システム (AI Camera Auto Inspection System)

**バージョン**: 3.3.1  
**最終更新**: 2026-04-06

## 📌 これは何をするプログラム？
このプログラムは、**カメラとAI（人工知能）を使って、部品の欠品や個数を自動で検査するシステム**です。
例えば工場の生産ラインなどで、「部品が正しく付いているか？」「指定された個数あるか？」を、人間の目の代わりにAIがカメラの映像を見て素早く判定します。

---

## 💡 どんなことができるの？

1. **AIによる自動判定 (YOLO)**
   * 事前に学習させたAIモデルを使って、カメラに映った物体（部品など）を見つけ出します。
   * 「犬が2匹、猫が1匹いるか？」といった複数条件の組み合わせ（AND条件）で精度の高いOK/NG判定が可能です。

2. **ハードウェアとの連携 (GPIO操作)**
   * **トリガー入力**: 外部のセンサーやボタンからの信号（トリガー）をキッカケに撮影と検査をスタートします。
   * **パターン切り替え**: 検査する製品の種類が変わった場合、スイッチ（セレクターピン）の切り替えで自動的に検査条件（パターン）が変わります。
   * **結果出力 (パトライト・ブザー)**: 検査結果が「OK」なら緑のランプ、「NG」なら赤のブザーを鳴らすといった外部機器の制御が可能です。

3. **証拠画像の自動保存と履歴管理**
   * 検査した瞬間の画像を自動で保存します。（保存先は自動で作られます）
   * 画面上に「最新のNG履歴」が表示され、ダブルクリックするだけでその時のエラー画像をすぐに確認できます。
   * 日々の検査結果はエクセルで開けるCSVファイルとして自動集計されます。

4. **パソコン(Windows)でもテスト可能（モックモード）**
   * 通常はラズベリーパイ（Raspberry Pi）などの専用機器の上で動かしますが、Windowsなどの普通のパソコンで起動した場合は「仮想パネル」が立ち上がり、ボタンやセンサーがなくても画面上のクリックだけで動作テストができます。

## 🔄 動作フロー (処理の流れ)

本システムは以下の流れで一連の検査を自動実行します。

### 1. 起動と初期設定
- アプリケーションを起動すると、画像やログを保存するためのフォルダ（`results` 等）が自動生成されます。
- 初期起動時は「設定」画面から、使用する**AIモデルのパス（.pt または ncnnフォルダ）**や、**プレビュー更新レート**、カメラごとの情報などを割り当てます。

### 2. パターン（検査条件）の登録
- 設定画面の「パターン設定」にて、検査のルールを作成します。
  - **判定条件 (AND)**: カメラ1では「ネジが4個」、カメラ2では「ラベルが1個」などのルールを定義します。
  - **ピン条件**: セレクターピン(GPIO)のON/OFFの組み合わせによって、どのパターンを実行するかを決定します。

### 3. メイン画面での待機・検査実行ループ
- 設定を完了してメイン画面に戻ると**トリガー待機状態**になります。
- 外部機器からトリガーピン(GPIO)に信号が入ると、以下のサイクルが実行されます：
  1. **状態確認**: 対象のセレクターピンの状態を読み取り、実行する「パターン（検査条件）」を決定します。
  2. **撮影・推論**: 接続されている全カメラの最新フレームを同時に取得し、AIモデル（YOLO等）に入力して物体検出を行います。
  3. **条件合致チェック**: 検出された物体のクラス（種類）や個数が、設定したパターンの条件をすべて満たしているかを判定します。
  4. **結果保存と出力**:
     - 条件を満たしていれば「OK」、満たしていなければ「NG」となります。
     - それぞれ対応するGPIO出力ピン（OKピン/NGピン）が動作し、パトライトなどを点灯・ブザーを鳴動させます。
     - 判定結果の画像（検出枠付き）とログ（CSV）を指定フォルダへ自動保存します。

### 4. リトライ機能（NG時のみ）
- NG判定となった場合、設定された「最大リトライ回数」まで自動的に再撮影と再推論を繰り返します。これにより、ワークの揺れや一時的な認識漏れによる誤判定を防ぎます。

## 🚀 おすすめの使い方

アプリ用画面には大きく分けて**「検査モード」**と**「撮影モード」**の2つがあります。

### ステップ1: AIに勉強させるための画像をたくさん撮る（撮影モード）
最初はAIが賢くないため、設定画面から「撮影モード」に切り替えます。
部品が流れてきてトリガーが入るたびに、判定はせずに画像をたくさん保存してくれます。この画像を使ってAIモデルを作成します。

### ステップ2: 検査のルールを決める（設定画面）
「設定」ボタンから、カメラの設定や「何を何個見つけたらOKとするか？」というルール（パターン）を作成します。
**Raspberry Pi環境で動作が重い場合**は、「システム設定」タブから「プレビュー更新レート」を `0.5` fps など低く設定することで、負荷を大きく下げることができます。

### ステップ3: 実際の検査をスタート（検査モード）
「検査モード」に切り替えて待機します。
部品が流れてきてセンサーが反応すると、一瞬でカメラが撮影し、AIがルール通りか確認してOK/NGを出してくれます！

## 🛠️ 起動方法

### Windowsでテスト実行する場合（仮想パネルが起動します）
コマンドプロンプトやPowerShellを開き、このフォルダの中で以下のコマンドを実行します。
```cmd
# 必要なライブラリを入れる（初回のみ）
pip install -r code/inspection_app/requirements.txt

# アプリを起動する
python code/inspection_app/main.py
```

### Linux (Raspberry Piなど) で実行ファイルを作る場合
Pythonがよくわからない人でもダブルクリック感覚で起動できるように、実行ファイルを作るスクリプトが用意されています。
```bash
cd code
chmod +x build.sh
./build.sh
```
完了すると `dist/inspection_app` フォルダの中に実行ファイルが完成します。
